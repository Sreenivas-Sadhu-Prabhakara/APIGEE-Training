Capstone brief

Build a banking accounts API proxy that ties the whole course together: REST resource design, Apigee X environment setup, proxy and target endpoints, shared flows, low/medium/high security tiers, product governance, CI/CD, observability, negative testing, and an explicit rollback plan.

Architecture decision record

# ADR: Security tier for /banking/v1/accounts

Decision: High-security tier.

Context:
- Endpoint returns regulated account data.
- Callers are third-party providers.
- Consent and audit evidence are mandatory.

Controls:
- OAuth authorization flow with PKCE at the authorization server.
- JWT validation at Apigee resource gateway.
- Sender-constrained token strategy documented.
- Consent ID, resource scope, quota, spike arrest, and audit headers.

Consequences:
- More integration work for partners.
- Stronger replay resistance, traceability, and operational evidence.
