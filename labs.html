Environment prerequisites

The labs include a complete Apigee X setup day. By the end of that day every learner team should have a verified Google Cloud project, Apigee organization, environment, environment group, hostname, deploy permission, and shell profile.

Required

Apigee org, env, env group, deployed runtime, API proxy deploy permission.

Recommended

Shared learner project, Cloud Logging sink, service account per team, CI runner.

Optional

Private CA, mock OIDC provider, JWKS endpoint, and developer portal sandbox.

Apigee X setup day

Provision once, verify many times.

Use the Google Cloud console path for beginners and the command-line path for advanced teams. The point of the day is not speed; it is understanding what each platform component does.

1Project

Billing, APIs, region, IAM, and learner service accounts.

2Org

Apigee X evaluation or paid organization provisioning path.

3Runtime

Environment, environment group, hostname, instance, and attachment.

4Access

DNS/TLS path, deploy roles, token access, and test client.

5Evidence

Verification commands, screenshots, and setup decision record.

gcloud auth login
gcloud config set project "$PROJECT_ID"

  cloudresourcemanager.googleapis.com serviceusage.googleapis.com

3. Minimal proxy endpoint

<ProxyEndpoint name="default">
  <PreFlow name="PreFlow">
    <Request>
      <Step><Name>SA-Burst-Protection</Name></Step>
      <Step><Name>AM-Set-Request-Headers</Name></Step>
    </Request>
    <Response/>
  </PreFlow>
  <Flows>
    <Flow name="GET accounts">
      <Condition>(proxy.pathsuffix MatchesPath "/accounts") and (request.verb = "GET")</Condition>
    </Flow>
  </Flows>
  <HTTPProxyConnection>
    <BasePath>/banking/v1</BasePath>
  </HTTPProxyConnection>
  <RouteRule name="default">
    <TargetEndpoint>default</TargetEndpoint>
  </RouteRule>
</ProxyEndpoint>

4. Minimal target endpoint

6. Deploy command

mvn clean install -Ptest \
  -Dorg="$APIGEE_ORG" \
  -Denv="$APIGEE_ENV" \
  -Dtoken="$TOKEN" \
  -Dapigee.options=update
