High, medium, and low security

Pattern library for real API portfolios

The goal is not one universal security stack. Learners practice choosing proportional controls, proving the decision, and knowing when a lower tier is no longer acceptable.

High

Open banking style financial API

For account, payment, consent, regulated data, and high-risk partner APIs where replay, token theft, repudiation, and regulatory audit matter.

  • OAuth 2.0 authorization code with PKCE; FAPI 2.0 concepts.
  • PAR and signed request/response patterns where ecosystem requires them.
  • Sender-constrained tokens through mTLS or DPoP design choice.
  • Consent binding, account/resource authorization, idempotency, and tamper-evident audit.
Medium

External partner API

For B2B APIs where the partner is known, data is important, and controls should prevent abuse without full open banking ceremony.

  • Client credentials, external JWT, or opaque token introspection.
  • API products, scopes, quotas, spike arrest, and partner-specific analytics.
  • Schema validation, threat protection, and clear error contracts.
  • Certificate pinning or mTLS when partner connectivity justifies it.
Low

Internal or public low-risk API

For read-only, non-sensitive, low-blast-radius APIs where speed and simplicity matter, but gateway hygiene still matters.

  • API key or network-level trust plus basic quota.
  • CORS, payload size limits, cache strategy, and friendly errors.
  • Minimal identity propagation and no sensitive data logging.
  • Clear promotion path to medium if data sensitivity changes.

FAPI-style context validation

var requiredClaims = ["iss", "sub", "aud", "exp", "iat", "scope", "client_id"];
var missing = requiredClaims.filter(function (claim) {
  return !context.getVariable("jwt.JWT-Verify-Partner-Token.claim." + claim);
});

var scopes = String(context.getVariable("jwt.JWT-Verify-Partner-Token.claim.scope") || "");
var hasAccountsRead = scopes.split(" ").indexOf("accounts:read") !== -1;
var consentId = context.getVariable("request.header.x-consent-id");

if (missing.length || !hasAccountsRead || !consentId) {
  context.setVariable("security.failure.reason", missing.join(",") || "scope_or_consent");
  throw new Error("High-security context validation failed");
}
